## Overview

OmniClaw operates strictly on **Triggers and Events** routed through the **8 Core Daemons Neural Bus**.
There are NO virtual -departments- or -Corp modes-. All workflows are deterministic data pipelines.

---

## TRIGGER MAP: THE OMNICLAW NEURAL PIPELINE

All major operations trigger a cascading Sequence of Events via the Core Daemons.

### 1. Ingestion Event (Raw Data Arrival)
**Trigger:** Raw repo is fetched, a user drags a file, or external webhook hits the system.
1. **[OIW]** (Intake Workflow) captures the data and places it in `$OMNICLAW_ROOT/vault/tmp/`.
2. **[OSF]** (Sandbox Firewall) scans the raw contents for API keys, sensitive data, and malicious payload traces.
3. If OSF passes, it alerts **[OAP]** (Assimilation Pipeline).

### 2. Processing Event (Assimilation & Healing)
**Trigger:** OAP receives green light from OSF via the Neural Bus.
1. **[OAP]** extracts metadata, compiles schemas, and determines the target ecosystem location.
2. **[OHD]** (Health Daemon) is called by OAP to inject YAML frontmatter, fix markdown links, and ensure formatting strictly adheres to OmniClaw standards.
3. OAP executes the *Forge in-place* move to place the finalized asset directly into `ecosystem/` or `brain/knowledge/`.

### 3. Registry & Mapping Event (Finalization)
**Trigger:** A new Skill or Knowledge file lands in the Ecosystem.
1. **[OER]** (Entity Registrar) detects the new validated file and writes its metadata into `brain/registry/SKILL_REGISTRY.json`.
2. **[OMA]** (Master Architect) sweeps the directory and updates `OMA_SYSTEM_MAP.json` to reflect the new boundaries.
3. **[OA]** (Academy) optionally analyzes the trash or logs left in `vault/tmp` for structural lessons before OHD incinerates them.

---

## BACKGROUND TRIGGERS (IDLE STATE)

**Trigger:** The system enters an idle loop (No active LLM prompt).
- **OMA Background Tick:** OMA performs SHA-256 hash checks on directories. If hashes change without OER registry updates, it quarantines the file.
- **OHD Janitor Tick:** OHD sweeps `vault/tmp/logs` and incinerates files older than 48 hours.

---

## INVOCATION PROTOCOLS

When the Human User (CEO) prompts the Central Agent (Gemini/Claude):
1. Agent checks `OMA_SYSTEM_MAP.json` to understand the environmental boundaries.
2. Agent checks `SKILL_REGISTRY.json` to equip necessary functional tools (Skills).
3. The Agent CANNOT bypass OSF or OER. Any code generated by the Agent must also abide by the Vault Enclosure rule.
