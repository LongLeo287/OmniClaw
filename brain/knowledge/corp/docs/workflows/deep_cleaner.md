# OmniClaw Deep Sanitation Workflow

## OS-Level Daemon (Dept 22 - Operations)

The Deep Cleaner is an aggressively enforced software sanitizer integral to the OmniClaw operating system. It operates under the Zero-Trust Architecture to ensure that no ephemeral secrets, dangling cache files, or orphaned databases are left exposed in the workspace.

### Core Functions

1. **SQLite & DB Purge:** Destroys abandoned `.sqlite` and database cache remnants unless they are explicitly protected within the `/storage/vault/` architecture.
2. **Git Footprint Eradication:** Sweeps broken Git repositories, lock files, and accidental bare-clones.
3. **Ghost Cache Clearing:** Erases dangling temporary `.cache` files across the entire OS directory tree generated by Python, Node, LLMs, or the browser MCPs.
4. **Log Trimming:** Archives or wipes excessive log outputs from agents (e.g., Nova, Strix) to prevent log-bloat and accidental API key logging.

### Execution

To run the Deep Cleaner daemon manually, execute the following script from the root workspace directory:

```ps1
python system/ops/scripts/omniclaw_deep_cleaner.py
```

### Safety Interlocks

* **Vault Protection:** The cleaner intrinsically bypasses the `/storage/vault/` path, creating an absolute locale protective lock around user data and persistent state files.
* **Brain Sovereignty:** The data stored in `/brain/corp/` and `/brain/registry/` is verified against the `MASTER_INDEX.md` and cannot be swept away by this script, keeping OmniClaw intelligent and secure.
