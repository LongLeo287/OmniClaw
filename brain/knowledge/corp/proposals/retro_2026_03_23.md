## Lessons Learned

### Lesson 1: "Rule synchronization MUST be done for both agents"
- **Happened:** Rule 3-Tier initially only added GEMINI.md, forgot CLAUDE.md
- **Distilled fact:** Any new Rules in GEMINI.md must be synchronized to CLAUDE.md within the same session. Don't let it go to the next session.
- **Applied to:** All agents + Antigravity boot sequence

### Lesson 2: "Files scattered around = Wrong architecture"
- **Happens:** I create .cursorrules, .windsurfrules, .agentrules scattered throughout the root directory
- **Distilled fact:** OmniClaw has a clear structure. Agent X's new rules must GO INTO agent X's boot file, not creating separate new files.
- **Applied to:** Antigravity, Claude Code, all future agents

### Lesson 3: "REJECT ≠ learn nothing"
- **Occurs:** Initially REJECT graphrag, agency-agents, all-agentic-architectures because of "duplicate functions"
- **Distilled fact:** Duplicate repo tool ≠ useless repo. Must read to find patterns/concepts that are not yet available in OmniClaw, cherry-pick knowledge/notes, then decide not to use the tool.
- **Applied to:** Dept 20 (CIV), Dept R&D, repo-evaluation.md workflow

### Lesson 4: "Processes must be mapped to departments"
- **Happened:** After creating new workflows (repo-evaluation, plugin-lazy-load), forgot to update the Memory of related departments
- **Distilled fact:** Each new workflow MUST be mapped to the corresponding Department Memory DURING THE SAME SESSION. Workflow without department owner = dead workflow.
- **Applied to:** Dept 4, Dept 10, Dept 20

### Lesson 5: "Post-session must run properly"
- **Occurs:** Many sessions end without running corp-learning-loop
- **Distilled fact:** After each session with system changes, it is REQUIRED to run: RETRO → Memory Update → Blackboard Update → Proposals
- **Applied to:** Antigravity (all sessions)
